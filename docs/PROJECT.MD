# Real-ESRGAN Video Web App

基于 Real-ESRGAN 的高性能视频/图像超分辨率增强 Web 应用。

## 🧠 开发哲学 (Fail-Fast Principle)

本项目严格遵循 **“快速失败”** 原则：
1.  **禁止静默失败**：严禁在代码中使用 `try-except pass` 来忽略错误。
2.  **强制异常抛出**：如果配置缺失、环境异常或核心库加载失败，应用必须立即终止运行并抛出详细的错误日志。
3.  **无默认回退**：除非用户明确要求并配置，否则不允许在代码中私自编写“退而求其次”的容错逻辑（如：VRAM 检测失败后私自回退到 CPU 处理且不报错），这会掩盖环境配置问题。
4.  **显式依赖**：所有依赖必须在 `Dockerfile` 中显式安装，不使用非标准的运行时 hack。

## 🛠️ SOP 标准操作流程

任何代码变更必须遵循以下顺序，严禁越级：
1.  **审计与计划**：由 AI 发现问题或收到需求，提交技术分析及修改计划，等待批准。
2.  **本地开发**：在本地工作区完成代码修改。
3.  **Git 提交**：**任何**代码或配置变动必须执行 `git add` 和 `git commit`。
4.  **本地验证 (Verified Build)**：在本地构建测试镜像，运行并检查日志（确保功能正常且 Fail-Fast 逻辑有效）。
5.  **推送申请**：验证通过后，申请执行推送。
6.  **生产同步**：推送完成后，同步更新正式部署环境。
